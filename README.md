# CastHub Parser

Приложение для автоматического сбора объявлений о кастингах и вакансиях из Telegram-чатов. Парсер читает сообщения, обрабатывает их с помощью ИИ и публикует структурированные объявления на app.casthub.kz.

---

## Требования

- **Windows 10/11** (64-bit)
- **Node.js 20+** — скачать с https://nodejs.org (выбрать LTS версию, при установке оставить все галочки по умолчанию)
- **Git** — скачать с https://git-scm.com/download/win (при установке оставить всё по умолчанию)
- **OpenAI API ключ** — получить на https://platform.openai.com/api-keys
- **Telegram API credentials** — получить на https://my.telegram.org

## Установка (один раз)

### 1. Скачать проект

Откройте **Командную строку** (нажмите `Win+R`, введите `cmd`, нажмите Enter).

Введите команды по одной, нажимая Enter после каждой:

```
cd C:\
git clone <URL репозитория> casthub-parser
cd casthub-parser
npm install --legacy-peer-deps
```

Дождитесь завершения установки (может занять 2-3 минуты).

### 2. Собрать приложение

```
npx electron-vite build
```

### 3. Пересобрать нативные модули

```
npx @electron/rebuild
```

Если появится ошибка про Visual Studio — это нормально, приложение будет работать без этого шага.

## Запуск

### Вариант 1: Через bat-файл (рекомендуется)

Найдите в папке проекта файл `dev.cmd` и дважды кликните на него. Откроется окно терминала и через несколько секунд появится окно приложения.


## Первоначальная настройка

После запуска откроется окно приложения. Слева — меню навигации. Начните с раздела **Настройки**.

### Шаг 1: Настройки → CastHub API

| Поле | Что вводить |
|------|------------|
| **URL** | `https://app.casthub.kz` (уже заполнено) |
| **API ключ** | Ключ для связи с сайтом. Его выдаёт разработчик. Это длинная строка из букв и цифр |

После ввода нажмите **Сохранить**.

### Шаг 2: Настройки → ИИ-провайдер

| Поле | Что вводить |
|------|------------|
| **Провайдер** | Оставьте `OpenAI` |
| **API ключ** | Ваш ключ OpenAI. Как получить — см. ниже |
| **Модель** | `gpt-4o-mini` (уже заполнено, самая дешёвая) |

#### Как получить OpenAI API ключ:
1. Зайдите на https://platform.openai.com
2. Зарегистрируйтесь или войдите
3. Перейдите в https://platform.openai.com/api-keys
4. Нажмите **"Create new secret key"**
5. Скопируйте ключ (начинается с `sk-...`)
6. Вставьте в поле "API ключ" в настройках парсера

> **Важно:** Ключ показывается только один раз! Сохраните его в безопасном месте. На счету OpenAI должен быть баланс (минимум $5). Стоимость обработки одного объявления — примерно $0.001-0.005.

### Шаг 3: Настройки → Автоматизация

| Поле | Что выбрать | Зачем |
|------|------------|-------|
| **Авто-публикация** | **ВЫКЛ** (серый) для начала | Объявления сначала попадут в очередь, вы проверите качество. Потом можно включить |
| **Срок действия (дни)** | `3` | Через сколько дней объявление исчезнет с сайта, если в нём не указана дата |
| **Кеш дедупликации (дни)** | `7` | Сколько дней помнить объявления чтобы не постить одинаковые |

Нажмите **Сохранить**.

### Шаг 4: Подключение Telegram

Перейдите в раздел **Чаты** (в левом меню).

#### Получение API credentials:
1. Зайдите на https://my.telegram.org с того аккаунта, который будет мониторить чаты
2. Введите номер телефона → получите код → войдите
3. Нажмите **"API development tools"**
4. Если приложение ещё не создано — заполните:
   - App title: `CastHub Parser`
   - Short name: `casthubparser`
   - Нажмите "Create application"
5. Скопируйте два значения:
   - **App api_id** — число (например `12345678`)
   - **App api_hash** — строка (например `0a1b2c3d4e5f6a7b8c9d0e1f`)

#### Подключение в парсере:
1. Вставьте **API ID** и **API Hash** в соответствующие поля
2. Нажмите **"Подключить"**
3. Введите **номер телефона** (в формате +7XXXXXXXXXX)
4. Введите **код** из Telegram (придёт в приложение Telegram или SMS)
5. Если включена **двухфакторная аутентификация** — введите пароль

Индикатор "Telegram" в левом нижнем углу станет **зелёным** — значит подключено.

> **Важно:** При следующих запусках Telegram подключится автоматически — вводить код заново не нужно.

### Шаг 5: Добавление чатов

После подключения Telegram:

1. Нажмите **"+ Добавить чат"**
2. Откроется список всех ваших групп и каналов в Telegram
3. Найдите нужный чат (можно использовать поиск)
4. Кликните на него — он добавится в список мониторинга
5. Укажите дату **"Собирать с"** — с какого числа загрузить старые сообщения
6. Нажмите **"Синхр."** чтобы загрузить историю

Повторите для всех нужных чатов.

## Ежедневное использование

### Что происходит автоматически

После настройки парсер работает сам:
- Новые сообщения в отслеживаемых чатах подхватываются **в реальном времени**
- Каждое сообщение проверяется ИИ — это объявление о кастинге/вакансии или нет
- Если да — ИИ извлекает структурированные данные (роли, контакты, город, даты)
- Объявление попадает в **Очередь**

### Модерация (если авто-публикация выключена)

1. Перейдите в раздел **Очередь**
2. Объявления со статусом **"На модерации"** ждут вашего решения
3. Кликните на объявление чтобы раскрыть его
4. Слева — оригинальный текст, справа — что извлёк ИИ (JSON)
5. Кнопки:
   - **Одобрить** — отправить на сайт
   - **Перепроверить** — заново прогнать через ИИ (если результат неточный)
   - **Отклонить** — удалить, не публиковать

### Доставка на сайт

- Одобренные объявления попадают в статус **"Ожидает"**
- На дашборде нажмите **"Запустить доставку"** — объявления автоматически отправятся на сайт каждые 30 секунд
- Или нажмите **"Доставить сейчас"** для немедленной отправки
- Успешно доставленные получают статус **"Доставлено"**

### Если авто-публикация включена

Объявления пропускают этап модерации и сразу отправляются на сайт. Доставка запускается автоматически при старте приложения.

## Решение проблем

### Белый экран при запуске
Запустите через командную строку с командой `set ELECTRON_RUN_AS_NODE=` перед `npx electron-vite dev`.

### Telegram не подключается
- Проверьте что API ID и API Hash правильные (https://my.telegram.org)
- Убедитесь что номер телефона в формате +7XXXXXXXXXX
- Если ошибка — закройте приложение, удалите папку `tdlib_db` в `%APPDATA%\casthub-parser\` и попробуйте заново

### Объявления не появляются в очереди
- Проверьте что Telegram подключён (зелёный индикатор)
- Проверьте что чат добавлен в мониторинг (раздел "Чаты")
- Посмотрите раздел "Логи" — там видно все входящие сообщения и причины пропуска
- Сообщения короче 20 символов автоматически пропускаются

### Ошибка "Unauthorized" при доставке
- Проверьте API ключ CastHub в настройках
- Убедитесь что ключ совпадает с тем что на сервере

### Ошибка ИИ при обработке
- Проверьте что OpenAI API ключ введён и на счету есть баланс
- Нажмите "Перепроверить" на объявлении с ошибкой
- Если ошибки повторяются — проверьте промпты в настройках

### Настройки не сохраняются
- Нажмите кнопку **"Сохранить"** после ввода
- Для полного применения изменений нажмите **"Перезапустить"**

## Стоимость использования

Парсер использует OpenAI API для обработки каждого сообщения. Примерная стоимость:

| Действие | Стоимость |
|----------|----------|
| Проверка одного сообщения (релевантно или нет) | ~$0.0002 |
| Полная обработка одного объявления (5 шагов) | ~$0.002-0.01 |
| 100 объявлений в день | ~$0.20-1.00 |
| 1000 объявлений в день | ~$2.00-10.00 |

Нерелевантные сообщения (болтовня, мемы) проверяются только на первом шаге и стоят в ~10 раз дешевле.

## Техническая информация

- **Стек:** Electron 28 + React 19 + TypeScript + Tailwind CSS
- **Telegram:** TDLib через tdl + prebuilt-tdlib (userbot)
- **AI:** OpenAI GPT-4o-mini (сменяемый провайдер)
- **Хранилище:** SQLite (sql.js) в `%APPDATA%\casthub-parser\`
- **Данные:** настройки, очередь, промпты, Telegram-сессия — всё локально на машине

### Файлы данных

Все данные хранятся в `%APPDATA%\casthub-parser\`:

| Файл/Папка | Содержимое |
|------------|-----------|
| `casthub-parser.db` | База данных (настройки, очередь, промпты) |
| `tdlib_db/` | Сессия Telegram (не удалять, иначе придётся заново авторизоваться) |
| `tdlib_files/` | Кеш файлов Telegram |

### Безопасность

- Telegram-сессия хранится локально, никуда не передаётся
- API ключи хранятся в локальной SQLite базе
- Текст объявлений проходит санитизацию перед обработкой (защита от SQL и prompt injection)
- Все проекты на сайте помечены: "Получено из открытых источников · Материал обработан с использованием ИИ"
